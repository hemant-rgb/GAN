# Vanilla GAN for Face Generation

> **Important:** This repository contains only a **small subset (1,000 images)** of the CelebA dataset to keep the repository lightweight. For proper training and better image generation results, **download the complete CelebA dataset** from the official source and replace the provided image folder with the full dataset before training.

---

## Overview

This project implements a **Vanilla Generative Adversarial Network (GAN)** using **PyTorch** to generate realistic human face images from random noise. The model is trained on the **CelebA (CelebFaces Attributes)** dataset and demonstrates the fundamental adversarial learning process introduced in the original GAN paper by Goodfellow et al.

Unlike modern GAN variants such as DCGAN, WGAN, or StyleGAN, this implementation focuses on the **basic GAN architecture**, making it ideal for understanding the core concepts behind generative models.

---

## Demo

After training, the generator learns to transform random latent vectors into realistic-looking face images.

Example workflow:

* Load and preprocess CelebA images.
* Train the Generator and Discriminator simultaneously.
* Generate new faces from random noise.
* Save generated samples after regular training intervals.

# Vanilla GAN Architecture

A Vanilla GAN consists of two neural networks that compete against each other:

## 1. Generator (G)

The Generator learns to create fake images that resemble real images.

**Input:**

* Random noise vector (latent vector)

**Output:**

* Synthetic face image

Its objective is to fool the Discriminator into believing the generated image is real.

```
Random Noise (z)
        │
        ▼
   Generator
        │
        ▼
 Generated Image
```

---

## 2. Discriminator (D)

The Discriminator acts as a binary classifier.

It receives both:

* Real images from the dataset
* Fake images generated by the Generator

Its task is to predict whether an image is **Real** or **Fake**.

```
               Real Image ───────►
                                  │
                                  ▼
                           Discriminator
                                  ▲
                                  │
 Generated Image ────────────────►

Output:
Real → 1
Fake → 0
```

---

## Adversarial Training

Training follows a two-player minimax game:

### Step 1 — Train the Discriminator

The Discriminator learns to:

* Classify real images as **1**
* Classify fake images as **0**

Loss:

```
Real Images  -> Target = 1
Fake Images  -> Target = 0
```

---

### Step 2 — Train the Generator

The Generator produces fake images and attempts to fool the Discriminator.

Instead of wanting fake images classified as **0**, the Generator wants them classified as **1**.

```
Noise
   │
   ▼
Generator
   │
   ▼
Fake Image
   │
   ▼
Discriminator
   │
Target = 1
```

As training progresses:

* The Generator creates increasingly realistic images.
* The Discriminator becomes better at detecting fake images.
* Eventually, both networks reach an equilibrium where generated images closely resemble the training data.

## Training Pipeline

```
Noise Vector
      │
      ▼
 Generator
      │
 Fake Images
      │
      ▼
 Discriminator ◄──────── Real Images
      │
      ▼
  BCE Loss
      │
      ▼
 Update Networks
```
